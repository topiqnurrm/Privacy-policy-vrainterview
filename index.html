Aplikasi VRAINTERVIEW dikembangkan sebagai proyek penelitian berbasis Virtual Reality untuk simulasi wawancara kerja yang imersif dan interaktif. Aplikasi ini memerlukan koneksi internet dan menggunakan layanan Firebase serta Google Gemini API. Dengan menggunakan aplikasi ini, Anda menyetujui kebijakan privasi yang tercantum di bawah ini.
01

Informasi yang Dikumpulkan

VRAINTERVIEW mengumpulkan beberapa kategori data yang diperlukan untuk menjalankan fitur simulasi wawancara, fitur persiapan wawancara, dan fitur manajemen akun secara penuh. Berikut rincian data yang dikumpulkan beserta tujuannya:

Data Akun & Identitas

EmailDigunakan untuk autentikasi akun melalui Firebase Authentication, baik login manual maupun login via Google. Email disimpan di Firebase dan menjadi satu-satunya data identitas yang tersimpan di server.
Nama PenggunaNama yang diinputkan saat pendaftaran atau diambil dari akun Google, disimpan di Firebase Firestore untuk keperluan personalisasi tampilan aplikasi.
Foto ProfilDiambil otomatis dari akun Google apabila pengguna melakukan login melalui Google. Foto hanya ditampilkan di halaman profil dan tidak disimpan ulang di server aplikasi.

Data Persiapan & Sesi Wawancara

CV / ResumeFile CV dalam format PDF dapat diunggah secara opsional, namun wajib diunggah agar pengguna dapat memulai sesi VR Interview. Isi teks CV diekstrak dan disimpan di Firebase Firestore sebagai konteks untuk menghasilkan pertanyaan wawancara yang relevan dengan profil pengguna.
Input SuaraJawaban wawancara direkam melalui mikrofon selama sesi VR Interview berlangsung. Rekaman diproses menjadi transkrip teks yang disimpan di Firebase Firestore sebagai catatan sesi. Selain itu, fitur persiapan wawancara juga menggunakan input suara untuk latihan menjawab pertanyaan.
Jawaban Sesi VRSeluruh jawaban atas 8 pertanyaan utama dan hingga 3 pertanyaan tambahan dalam satu sesi VR disimpan di Firebase Firestore untuk keperluan evaluasi AI dan riwayat wawancara pengguna.
Data PersiapanHasil dari fitur persiapan wawancara — termasuk analisis CV, hasil generate pertanyaan acak, contoh sesi interview lengkap, dan data latihan jawaban — disimpan di Firebase Firestore agar dapat diakses kembali tanpa perlu generate ulang setiap sesi.

Data Personalisasi (Bukan Data Pribadi)

Nama PerusahaanInput teks untuk menyesuaikan konteks wawancara. Tidak terikat pada identitas pengguna secara personal.
Posisi JabatanInput teks untuk menyesuaikan jenis pertanyaan yang dihasilkan AI serta relevansi analisis CV.
Bahasa & ModePreferensi bahasa wawancara (Bahasa Indonesia atau Inggris) dan mode wawancara yang digunakan untuk konfigurasi sesi.
Konteks TambahanInformasi konteks perusahaan dan posisi yang diinputkan secara opsional oleh pengguna untuk membantu AI menghasilkan pertanyaan yang lebih spesifik.
Pengaturan AplikasiPreferensi tampilan VR (offset jarak layar), pilihan mode TTS (Google atau bawaan perangkat), serta preferensi bahasa antarmuka, disimpan di Firebase Firestore dan perangkat lokal.

Tidak ada data sensitif seperti nomor telepon, lokasi GPS, informasi keuangan, atau data biometrik yang dikumpulkan maupun disimpan oleh aplikasi ini.

02

Penggunaan Informasi

Data yang dikumpulkan hanya digunakan untuk tujuan berikut:

Autentikasi AkunEmail digunakan untuk proses login dan manajemen akun melalui Firebase Authentication, termasuk reset kata sandi dan penghapusan akun.
Simulasi Wawancara VRCV dan data personalisasi dikirimkan ke Google Gemini API untuk menghasilkan 8 pertanyaan wawancara yang spesifik dan relevan dengan profil pengguna, beserta sesi pertanyaan tambahan kepada perusahaan.
Fitur Persiapan WawancaraCV dan data personalisasi digunakan oleh Google Gemini API untuk menghasilkan analisis CV (requirements, kesesuaian, saran perbaikan), pertanyaan acak latihan, saran jawaban, dan contoh sesi interview lengkap sebagai panduan persiapan sebelum sesi VR.
Evaluasi & RiwayatJawaban sesi disimpan agar pengguna dapat meninjau kembali riwayat wawancara, mengunduh laporan PDF, dan memantau perkembangan kemampuan mereka dari waktu ke waktu.
Text-to-Speech (TTS)Teks evaluasi hasil wawancara dibacakan menggunakan layanan Google TTS atau TTS bawaan perangkat sesuai preferensi pengguna, untuk menyampaikan hasil analisis secara audio di akhir sesi VR.
Penelitian AkademisData dianalisis secara anonim dan agregat untuk keperluan penelitian pengembang mengenai efektivitas simulasi wawancara berbasis VR dan kecerdasan buatan.

Kami tidak menggunakan data Anda untuk iklan, profiling komersial, atau menjualnya kepada pihak ketiga manapun.

03

Berbagi Informasi

VRAINTERVIEW tidak menjual, menukar, atau memindahkan informasi Anda kepada pihak luar. Data Anda hanya dibagikan kepada layanan pihak ketiga berikut yang diperlukan untuk menjalankan fungsi utama aplikasi:

Firebase (Google)Layanan backend untuk autentikasi, penyimpanan data sesi, data persiapan wawancara, pengaturan aplikasi, serta teks CV pengguna. Data diproses sesuai kebijakan privasi Google/Firebase.
Google Gemini APICV dan konteks sesi dikirimkan ke API ini untuk menghasilkan pertanyaan wawancara, analisis CV, saran jawaban, dan contoh sesi interview. Data tidak digunakan untuk melatih model oleh Google sesuai ketentuan layanan API.
Google TTS APITeks evaluasi dikirimkan ke layanan Google Text-to-Speech untuk menghasilkan audio pembacaan hasil analisis di akhir sesi VR, apabila pengguna memilih opsi Google TTS. Pengguna dapat memilih TTS bawaan perangkat sebagai alternatif tanpa mengirimkan data ke server Google.
Kewajiban HukumDiwajibkan oleh hukum atau perintah pengadilan yang berlaku di Indonesia.
Perlindungan PenggunaDiperlukan untuk melindungi hak, keselamatan, atau keamanan pengguna.
04

Keamanan Data

VRAINTERVIEW memerlukan koneksi internet untuk berfungsi, karena menggunakan infrastruktur cloud dari Firebase dan Google Gemini API. Kami menerapkan langkah-langkah keamanan berikut untuk melindungi data Anda:

Enkripsi TransitSemua komunikasi data antara aplikasi dan server menggunakan protokol HTTPS/TLS sehingga data terenkripsi selama pengiriman.
Firebase Security RulesAkses ke data di Firestore dan Storage dibatasi menggunakan Firebase Security Rules sehingga hanya pengguna yang terautentikasi yang dapat mengakses data miliknya sendiri.
Autentikasi AmanSistem login menggunakan Firebase Authentication yang mengelola keamanan kredensial secara terpusat tanpa menyimpan kata sandi secara langsung di database aplikasi.
Minimisasi DataHanya data yang benar-benar diperlukan untuk fungsi aplikasi yang dikumpulkan dan disimpan. Tidak ada data sensitif seperti nomor telepon, lokasi, atau informasi keuangan yang tersimpan.
Isolasi Data PenggunaSetiap data pengguna disimpan secara terpisah berdasarkan UID unik Firebase, sehingga tidak ada pengguna lain yang dapat mengakses data milik pengguna lain.

Meskipun kami menerapkan langkah-langkah keamanan terbaik, tidak ada sistem transmisi atau penyimpanan data yang 100% aman secara mutlak. Kami menyarankan pengguna untuk menjaga kerahasiaan kredensial akun mereka.

05

Hak Pengguna

Sebagai pengguna, Anda memiliki hak penuh atas data Anda:

Hak TahuMengetahui data apa saja yang dikumpulkan dan disimpan oleh aplikasi ini, sebagaimana dijelaskan secara rinci dalam kebijakan privasi ini.
Hak Hapus SesiMenghapus riwayat sesi wawancara satu per satu kapan saja langsung dari dalam aplikasi melalui halaman Laporan.
Hak Hapus CVMenghapus file CV dan data persiapan wawancara kapan saja melalui menu pengaturan profil di dalam aplikasi.
Hak Hapus AkunMenghapus akun beserta seluruh data terkait secara permanen dari Firebase, termasuk data autentikasi, profil, riwayat sesi, data persiapan, dan pengaturan aplikasi.
Cabut IzinMencabut izin akses mikrofon kapan saja melalui Pengaturan Android tanpa perlu menghapus akun. Mencabut izin mikrofon akan menonaktifkan fitur input suara.
Pilih Mode TTSMemilih metode text-to-speech yang digunakan, yaitu Google TTS (suara lebih natural, memerlukan koneksi internet) atau TTS bawaan perangkat (lebih cepat, tidak mengirim data ke server eksternal).
Hak BertanyaMengajukan pertanyaan atau keberatan mengenai privasi kepada pengembang melalui saluran kontak yang tersedia di bagian bawah halaman ini.
06

Hapus Akun & Data

Pengguna memiliki kendali penuh untuk menghapus data maupun akun mereka kapan saja, langsung dari dalam aplikasi VRAINTERVIEW. Proses penghapusan bersifat permanen dan tidak dapat dibatalkan.

Menghapus Riwayat Sesi Wawancara

Riwayat sesi wawancara dapat dihapus satu per satu melalui halaman Laporan. Tidak tersedia opsi hapus semua riwayat sekaligus — setiap sesi harus dihapus secara individual untuk memastikan pengguna tidak menghapus data yang masih dibutuhkan secara tidak sengaja.

1
Buka aplikasi VRAINTERVIEW dan masuk ke tab Laporan melalui navigasi bawah.
2
Pilih sesi wawancara yang ingin dihapus dari daftar riwayat yang tersedia.
3
Di halaman detail sesi, temukan opsi "Hapus Sesi" dan konfirmasi penghapusan.
4
Data sesi tersebut akan dihapus secara permanen dari Firebase Firestore. Ulangi langkah ini untuk setiap sesi yang ingin dihapus.

Menghapus File CV & Data Persiapan

Untuk menghapus file CV dan hasil persiapan wawancara tanpa menghapus akun:

1
Buka aplikasi dan masuk ke halaman Home, kemudian pilih menu edit CV.
2
Pilih opsi "Hapus CV" pada halaman pengaturan CV, kemudian simpan perubahan.
3
Data CV dan hasil analisis persiapan wawancara yang tersimpan akan dihapus secara permanen dari Firebase Firestore.

Menghapus Akun Secara Permanen

Penghapusan akun akan menghapus seluruh data Anda dari Firebase secara sekaligus, termasuk akun autentikasi, data profil, seluruh riwayat sesi wawancara, data persiapan wawancara, file CV, dan pengaturan aplikasi. Tidak diperlukan penghapusan data satu per satu terlebih dahulu.

1
Buka aplikasi dan masuk ke tab Profil melalui navigasi bawah.
2
Temukan dan pilih opsi "Hapus Akun" yang tersedia di halaman profil.
3
Baca informasi konfirmasi yang ditampilkan dengan seksama. Sistem akan meminta konfirmasi sebelum proses penghapusan dimulai.
4
Konfirmasi penghapusan. Akun dan seluruh data terkait akan dihapus secara permanen dan bersamaan dari Firebase Authentication dan Firestore.

Penghapusan akun bersifat permanen dan tidak dapat dibatalkan. Seluruh riwayat sesi, data persiapan, CV, dan pengaturan akan ikut terhapus secara bersamaan. Pastikan Anda telah mengunduh atau menyimpan laporan yang diperlukan sebelum melanjutkan. Jika mengalami kendala, hubungi pengembang melalui kontak yang tersedia di bagian bawah halaman ini.
